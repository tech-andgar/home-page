Resolviendo Artefactos con `ImageFilter.blur` en Flutter en Android, iOS y macOS

Resolviendo Artefactos con ImageFilter.blur en Flutter en Android, iOS y macOS Link to heading

Introducción Link to heading

Al desarrollar aplicaciones móviles y de escritorio con Flutter, es común utilizar efectos de desenfoque para mejorar la interfaz de usuario. Uno de los métodos más utilizados es ImageFilter.blur. Sin embargo, algunos desarrolladores han encontrado artefactos visuales al aplicar este efecto en Android, iOS y macOS, mientras que en la web funciona correctamente. En este artículo, exploraremos este problema, entenderemos la razón detrás de él y proporcionaremos una solución efectiva.

El Problema Link to heading

Al aplicar ImageFilter.blur a un Container de color sólido (por ejemplo, una forma circular), aparecen artefactos visuales, como líneas horizontales y verticales como cruz, en las plataformas Android, iOS y macOS. Este problema no ocurre en la web, lo que sugiere que hay diferencias en cómo se maneja el efecto de desenfoque en diferentes plataformas.

Código que Reproduce el Problema Link to heading

import 'dart:ui';
import 'package:flutter/material.dart';

void main() {
  runApp(
    MaterialApp(
      home: Scaffold(
        body: Center(
          child: BlurredCircle(),
        ),
      ),
    ),
  );
}

class BlurredCircle extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ImageFiltered(
      imageFilter: ImageFilter.blur(sigmaX: 20, sigmaY: 20),
      child: Container(
        width: 200,
        height: 200,
        decoration: BoxDecoration(
          shape: BoxShape.circle,
          color: Colors.black.withOpacity(0.4),
        ),
      ),
    );
  }
}

Observación: Al ejecutar este código en Android, iOS o macOS, notarás una línea horizontal y vertical como cruz a través del círculo desenfocado. En la web, el círculo se renderiza correctamente sin artefactos.

Entendiendo la Causa Link to heading

Diferencia en Motores de Renderizado Link to heading

La razón principal detrás de este comportamiento es la diferencia en los motores de renderizado utilizados por Flutter en diferentes plataformas:

  • Web: Utiliza Skia como motor de renderizado.
  • Android, iOS y macOS: Utilizan Impeller, un nuevo motor de renderizado diseñado para mejorar el rendimiento y la consistencia en dispositivos móviles y de escritorio.

Esta diferencia en los motores de renderizado significa que ciertos efectos gráficos, como ImageFilter.blur, pueden comportarse de manera diferente en distintas plataformas.

Comportamiento de TileMode Link to heading

Por defecto, ImageFilter.blur utiliza TileMode.clamp. Este modo extiende los píxeles del borde del widget de forma infinita en todas las direcciones. Cuando se aplica a un color sólido, esto puede causar artefactos visuales, ya que los píxeles del borde (del mismo color) se extienden y generan líneas o discontinuidades.

En Impeller, este comportamiento se vuelve más evidente debido a cómo maneja los efectos de borde en comparación con Skia. Esto explica por qué el problema es visible en Android, iOS y macOS, pero no en la web.

La Solución: Usar TileMode.decal Link to heading

Para resolver este problema, necesitamos cambiar el parámetro tileMode de ImageFilter.blur a TileMode.decal. Este modo trata los píxeles fuera de los límites del widget como transparentes, evitando que los píxeles del borde se extiendan y eliminando los artefactos visuales.

Código Actualizado Link to heading

import 'dart:ui';
import 'package:flutter/material.dart';

void main() {
  runApp(
    MaterialApp(
      home: Scaffold(
        body: Center(
          child: BlurredCircle(),
        ),
      ),
    ),
  );
}

class BlurredCircle extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ImageFiltered(
      imageFilter: ImageFilter.blur(
        sigmaX: 20,
        sigmaY: 20,
        tileMode: TileMode.decal, // Agrega tileMode.decal aquí
      ),
      child: Container(
        width: 200,
        height: 200,
        decoration: BoxDecoration(
          shape: BoxShape.circle,
          color: Colors.black.withOpacity(0.4),
        ),
      ),
    );
  }
}

Resultado: Al ejecutar este código actualizado, el círculo desenfocado se renderiza correctamente en todas las plataformas sin ningún artefacto visual.

Explicación Detallada Link to heading

  • TileMode.clamp (Predeterminado): Extiende los píxeles del borde del widget, lo que puede causar artefactos cuando se aplica un desenfoque a un color sólido.
  • TileMode.decal: Trata los píxeles fuera del widget como transparentes, evitando la extensión de los píxeles del borde y eliminando los artefactos.

Al especificar tileMode: TileMode.decal, le indicamos al filtro de imagen que no extienda los píxeles del borde, lo que resulta en un desenfoque limpio y sin artefactos.

Beneficios de la Solución Link to heading

  • Elimina Artefactos Visuales: La imagen desenfocada se renderiza correctamente sin líneas ni discontinuidades.
  • Compatibilidad entre Plataformas: La solución funciona en Android, iOS, macOS y web.
  • Sencillez: Solo se requiere agregar un parámetro adicional a ImageFilter.blur.

Consideraciones Adicionales Link to heading

Futuras Actualizaciones de Flutter Link to heading

El equipo de Flutter ha indicado y Blog oficial de Flutter - Breaking changes que planea cambiar el valor predeterminado de tileMode a TileMode.decal en próximas versiones. Esto significa que, eventualmente, no será necesario especificar este parámetro para evitar los artefactos.

Entendiendo los Motores de Renderizado Link to heading

  • Skia: Es el motor de renderizado utilizado en Flutter Web. Maneja los efectos gráficos de cierta manera que evita los artefactos al aplicar ImageFilter.blur con los valores predeterminados.
  • Impeller: Es el motor de renderizado utilizado en Android, iOS y macOS. Está diseñado para mejorar el rendimiento, pero maneja los efectos de borde de manera diferente, lo que puede resultar en artefactos si no se ajusta tileMode.

Comprender estas diferencias es crucial para desarrollar interfaces de usuario consistentes en todas las plataformas.

Conclusión Link to heading

El uso de efectos de desenfoque en Flutter puede mejorar significativamente la experiencia del usuario, pero es importante entender cómo configurarlos correctamente para evitar problemas. Al especificar tileMode: TileMode.decal en ImageFilter.blur, podemos eliminar los artefactos visuales en Android, iOS y macOS, asegurando una interfaz consistente y de alta calidad en todas las plataformas.
